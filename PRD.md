# OHAYO!

## 1. 한 줄 정의
일본 문화와 운세에 관심이 많은 20대 일본어 초급 학습자가 별도의 공부 시간을 마련해야 한다는 부담 없이, 오늘의 일본식 별자리 운세를 읽는 과정에서 자연스럽게 일본어 단어를 학습하고 저장한 단어를 반복해서 복습할 수 있도록 돕는 모바일 웹 서비스.

---

## 2. 문제와 타깃

### 문제
20대 학습자는 전공 공부, 취업 준비, 아르바이트뿐 아니라 즐길 수 있는 콘텐츠도 많아 일본어 학습에 별도의 시간과 노력을 꾸준히 투자하기 어렵다.

기존 일본어 학습 앱이나 교재는 학습을 목적으로 직접 실행해야 하며, 반복적인 문제 풀이와 진도 중심의 방식으로 인해 흥미를 잃고 며칠 만에 중단하는 경우가 많다.

반면 운세 콘텐츠는 결과에 대한 궁금증을 바탕으로 매일 짧은 시간 동안 부담 없이 반복해서 확인한다. 따라서 운세를 확인하는 행동 안에 단어 학습을 결합하면 사용자가 별도의 학습을 시작하지 않아도 자연스럽게 일본어를 접하도록 만들 수 있다.

### 타깃
일본 문화와 일본 콘텐츠를 좋아하며, 재미와 흥미를 중심으로 가볍게 시작할 수 있는 공부 방식을 선호하는 20대 일본어 초급 학습자.

- SNS에서 일본식 별자리 운세를 자주 확인한다.
- 히라가나는 읽을 수 있지만 일본어 문장은 번역기에 의존한다.
- 일본어 학습 앱을 사용해 봤지만 반복적인 학습 방식에 흥미를 잃은 경험이 있다.
- 긴 학습보다 하루에 짧게 반복할 수 있는 콘텐츠를 선호한다.
- 학습한 단어를 저장해 나중에 다시 확인하고 싶어 한다.

## 3. 핵심 시나리오
운세를 확인한 다음 별도의 학습을 시작하게 하는 것이 아니라, 운세의 내용을 알아가는 과정 자체가 일본어 학습이 되도록 설계한다.

1. 사용자는 오늘의 12개 별자리 운세 순위를 확인한다.
2. 자신의 별자리를 선택해 상세 운세 화면으로 이동한다.
3. 한국어 해석 없이 일본어 운세 원문을 먼저 읽는다.
4. 운세 문장 안에 강조된 핵심 단어 3개를 확인한다.
5. 모르는 단어를 누르면 읽는 법과 한국어 뜻이 작은 카드로 표시된다.
6. 저장하고 싶은 단어의 저장 버튼을 누른다.
7. 로그인하지 않은 사용자는 Google 로그인 안내를 확인한다.
8. Google 로그인을 완료하면 기존 화면으로 돌아오며 선택했던 단어가 자동으로 저장된다.
8-1. 단어 저장에 성공하면 "단어가 저장되었어요" 토스트에 "바로 복습" 액션이 함께 표시된다. 선택하면 저장한 단어 화면으로 이동해 방금 저장한 단어부터 플래시카드로 복습할 수 있다. 선택하지 않으면 현재 보고 있던 운세 화면에 그대로 머문다. (PRD 개정)
9. 단어를 모두 확인한 뒤 전체 해석 보기를 눌러 일본어 원문과 한국어 해석을 비교한다.
10. 이후 저장 화면에서 저장한 단어를 플래시카드 형식으로 복습한다.
11. 같은 Google 계정으로 로그인하면 다른 기기에서도 저장한 단어를 확인할 수 있다.

## 4. 서비스 및 콘텐츠 원칙

### 학습 원칙
- 일본어 원문을 한국어 해석보다 먼저 노출한다.
- 사용자가 문장 안에서 모르는 단어를 직접 선택하도록 한다.
- 한 운세당 학습 단어는 정확히 3개만 제공한다.
- 긴 설명보다 읽는 법, 뜻, 문장 속 사용 예시를 중심으로 제공한다.
- 운세 확인과 단어 학습이 하나의 흐름 안에서 이어지도록 한다.

### 콘텐츠 원칙
- 오하아사 운세 페이지에 공개된 당일 별자리 순위와 일본어 운세 문장을 수집한다.
- 운세 순위와 원문의 내용은 AI가 새로 만들거나 수정하지 않는다.
- AI는 읽는 법, 한국어 번역, 핵심 단어 데이터 생성에만 사용한다.
- 화면에 운세의 원본 날짜와 출처를 표시한다.
- 오하아사 또는 ABC TV에서 직접 운영하는 공식 서비스처럼 표현하지 않는다.
- 이미지, 영상, 광고 콘텐츠는 수집하지 않는다.
- 로그인, CAPTCHA, 접근 제한이 있는 페이지를 우회하지 않는다.

### 출처 안내 문구
상세 운세 화면 하단에 다음 내용을 표시한다.

> 운세 출처: ABC TV 「おはよう朝日です」
> 일본어 학습을 위해 읽는 법과 한국어 해석을 추가했습니다.

추가로 다음 안내를 표시한다.

> OHAYO!는 ABC TV 및 「おはよう朝日です」의 공식 서비스가 아닙니다.

## 5. 기능 범위

- 오늘 날짜 표시
- 오늘의 12개 별자리 운세 순위 확인
- 별자리별 일본어 상세 운세 제공
- 일본어 원문 우선 노출
- 일본어 원문의 읽는 법 펼쳐보기
- 운세 문장 속 핵심 단어 3개 강조
- 강조된 단어 선택
- 단어의 읽는 법과 한국어 뜻 표시
- 단어 저장 및 저장 해제
- 전체 해석 보기를 통한 한국어 해석 공개
- 행운의 장소 또는 아이템 표시
- 운세 원본 날짜와 출처 표시
- 원문 페이지 이동 링크 제공

### Must · MVP 2: Google 로그인과 사용자별 저장
- Google OAuth 로그인
- 로그인 상태 확인
- 로그아웃
- 비로그인 상태에서 저장 버튼 선택 시 로그인 안내
- 로그인 완료 후 기존 운세 화면으로 복귀
- 로그인 전에 선택한 단어 자동 저장
- 사용자 계정별 단어 저장
- 동일 단어 중복 저장 방지
- 다른 기기에서 저장 데이터 동기화

### Must · MVP 3: 저장 단어 플래시카드 복습
- 저장한 단어 개수 확인
- 저장한 단어를 플래시카드 형식으로 확인
- 카드 앞면에 일본어 단어 표시
- 카드를 누르면 뒷면으로 전환
- 카드 뒷면에 읽는 법과 한국어 뜻 표시
- 단어가 사용된 일본어 운세 문장 표시
- 운세 문장의 한국어 해석 표시
- 단어 저장 날짜 표시
- 이전 카드와 다음 카드 이동
- 저장한 단어 삭제
- 저장한 단어가 없을 때 빈 화면 표시

### Must · M3 보완: 저장 성공 토스트의 바로 복습 액션 (PRD 개정)
- 단어 저장 성공 시 성공 토스트에 "바로 복습" 액션 표시
- "바로 복습" 선택 시 저장한 단어 화면으로 이동, 방금 저장한 단어부터 플래시카드로 복습
- 액션을 선택하지 않으면 현재 운세 상세 화면 유지 (자동 이동 없음)
- 이미 저장된 단어(중복)라면 "이미 저장된 단어예요" 토스트만 표시하고 액션은 없음

### Must · MVP 4: 오하아사 운세 수집
- 평일 운세 페이지 데이터 수집
- 토요일 운세 페이지 데이터 수집
- 운세 날짜 추출
- 12개 별자리 순위 추출

- 행운의 장소 또는 아이템 추출

- 검증된 데이터만 데이터베이스에 저장
- 수집 시각과 원본 URL 저장
- 하루 한 번 예약 실행
- 수집 실패 기록

## 6. 로그인 정책

### 비로그인 사용자가 할 수 있는 행동
- 오늘의 별자리 순위 확인
- 별자리별 상세 운세 확인
- 핵심 단어의 읽는 법과 뜻 확인
- 일본어 원문의 읽는 법 확인
- 전체 한국어 해석 확인
- 행운의 장소 또는 아이템 확인

### 로그인이 필요한 행동
- 단어 저장
- 저장한 단어 확인
- 저장한 단어 삭제
- 다른 기기에서 저장 데이터 불러오기

### 로그인 노출 시점
서비스에 처음 접속했을 때는 로그인을 요구하지 않는다.
사용자가 단어를 저장하려는 순간 다음 내용의 로그인 안내를 표시한다.

> 단어를 저장하려면 로그인이 필요해요.
> Google 계정으로 로그인하면 저장한 단어를 다시 복습하고 다른 기기에서도 이어서 학습할 수 있어요.

### 로그인 완료 후 처리
- 로그인 전에 보고 있던 운세 페이지를 임시로 저장한다.
- 로그인 전에 저장하려던 단어 ID를 임시로 저장한다.
- 로그인 완료 후 기존 운세 상세 화면으로 돌아온다.
- 로그인 전에 선택했던 단어를 자동으로 저장한다.
- 이미 저장된 단어라면 중복 저장하지 않는다.
- 로그인 완료 후 임시 데이터를 삭제한다.
- 로그인 취소 시 기존 운세 화면을 유지한다.

  - 한국어 뜻

## 8. 운세 데이터 수집 방식

### 전체 데이터 흐름
OHAYO!는 오하아사 별자리 운세 페이지에 공개된 당일 운세를 서버에서 하루 한 번 수집한다.

수집한 일본어 운세 원문은 Gemini API를 통해 일본어 읽는 법, 자연스러운 한국어 번역, 초급 학습자를 위한 핵심 단어 3개로 가공한다.

사용자가 서비스에 접속할 때마다 오하아사 페이지나 Gemini API를 호출하지 않고, 사전에 Supabase에 저장된 당일 데이터를 불러온다.

### 데이터 처리 순서
1. 매일 오전 오하아사 운세 페이지를 한 번 요청한다.
2. 페이지에서 운세 날짜를 확인한다.
3. 12개 별자리의 순위와 일본어 운세 문장을 수집한다.
4. 행운의 장소 또는 아이템을 함께 수집한다.
5. 수집 데이터가 올바른 형식인지 검증한다.
6. 검증된 원본 데이터를 Supabase에 저장한다.
7. Gemini API에 일본어 운세 원문을 전달한다.
8. 읽는 법, 한국어 번역, 핵심 단어 3개를 생성한다.
9. AI 응답의 JSON 구조와 단어 개수를 검증한다.
10. 검증을 통과한 학습 데이터를 Supabase에 저장한다.
11. 사용자는 데이터베이스에 저장된 운세를 조회한다.

- 대상 요일: 월요일~금요일

## 13. AI 동작 정의
AI는 크롤링이 완료되고 검증된 운세 원문에만 사용한다.
AI는 운세의 순위나 내용을 새로 생성하지 않는다.

### 입력
- 날짜
- 별자리 ID
- 일본어 별자리명
- 한국어 별자리명
- 별자리 순위
- 일본어 운세 원문
- 행운의 장소 또는 아이템
- 원본 페이지 URL

### AI 역할
- 일본어 운세 원문의 읽는 법 생성
- 자연스러운 한국어 번역 생성
- 원문 안에서 핵심 단어 3개 선정
- 단어의 기본형 생성
- 문장 속 활용형 추출
- 단어의 읽는 법 생성
- 문장 안에서 사용된 의미에 맞는 한국어 뜻 생성
- 일본어 원문을 선택 가능한 문장 조각으로 분리

### AI 출력
```json
{
  "date": "2026-07-10",
  "zodiacId": "taurus",
  "zodiacJapanese": "おうし座",
  "zodiacKorean": "황소자리",
  "rank": 1,
  "originalText": "実力を発揮するチャンス。大胆なアクションが幸運のカギ。",
  "readingText": "じつりょくを はっきする チャンス。だいたんな アクションが こううんの カギ。",
  "koreanTranslation": "실력을 발휘할 기회예요. 대담한 행동이 행운을 부르는 열쇠가 될 거예요.",
  "luckyItem": "スイーツ店",
  "sourceUrl": "https://example.com",
  "segments": [
    { "text": "実力", "vocabularyId": "taurus-20260710-01" },
    { "text": "を", "vocabularyId": null },
    { "text": "発揮する", "vocabularyId": "taurus-20260710-02" },
    { "text": "チャンス。", "vocabularyId": null },
    { "text": "大胆な", "vocabularyId": "taurus-20260710-03" },
    { "text": "アクションが幸運のカギ。", "vocabularyId": null }
  ],
  "vocabulary": [
    { "id": "taurus-20260710-01", "word": "実力", "surfaceForm": "実力", "reading": "じつりょく", "meaning": "실력" },
    { "id": "taurus-20260710-02", "word": "発揮する", "surfaceForm": "発揮する", "reading": "はっきする", "meaning": "발휘하다" },
    { "id": "taurus-20260710-03", "word": "大胆だ", "surfaceForm": "大胆な", "reading": "だいたんだ", "meaning": "대담하다" }
  ]
}
```

### 필드 정의
- `word`: 사전에서 찾을 수 있는 단어의 기본형
- `surfaceForm`: 실제 운세 문장에 등장한 형태
- `reading`: 단어의 히라가나 읽는 법
- `meaning`: 운세 문장 안에서 사용된 의미를 기준으로 한 한국어 뜻
- `segments`: 원문을 순서대로 조립할 수 있는 문장 조각
- `vocabularyId`: 선택 가능한 단어와 단어 데이터를 연결하는 ID

## 14. AI 프롬프트 규칙
- 한국인 일본어 초급 학습자를 돕는 친절한 일본어 교사 역할을 부여한다.
- 입력된 운세 원문의 내용을 변경하지 않는다.
- 새로운 운세 내용이나 문장을 추가하지 않는다.
- originalText는 입력받은 일본어 원문과 완전히 동일해야 한다.
- 원문 안에서 학습 가치가 높은 JLPT N5~N4 수준의 단어를 정확히 3개 선정한다.
- N5~N4 수준의 단어가 부족하면 초급 학습자가 알아두기 좋은 표현을 선정한다.
- 원문에 실제로 존재하는 단어만 선정한다.
- 조사, 접속사, 숫자, 단순 기호, 고유명사는 제외한다.
- 동사와 형용사의 word 값은 사전형으로 제공한다.
- surfaceForm에는 원문에 실제로 등장한 형태를 제공한다.
- 단어의 뜻은 해당 문장 안에서 사용된 의미를 기준으로 작성한다.
- 번역은 자연스럽게 작성하되 원문의 의미를 과장하거나 축소하지 않는다.
- 운세의 확정적인 표현을 임의로 강화하지 않는다.
- segments의 모든 text를 순서대로 연결했을 때 originalText와 정확히 일치해야 한다.
- 같은 문장 위치를 두 개 이상의 단어가 중복해서 사용하지 않도록 한다.
- 지정된 JSON 구조 이외의 설명을 출력하지 않는다.

## 15. AI 결과 검증
AI 응답을 데이터베이스에 저장하기 전에 다음 조건을 검증한다.

- JSON Schema와 일치하는가
- 필수 필드가 모두 존재하는가
- 입력한 날짜와 출력 날짜가 같은가
- 입력한 별자리 ID와 출력 별자리 ID가 같은가
- 입력한 순위와 출력 순위가 같은가
- originalText가 크롤링한 원문과 완전히 같은가
- 핵심 단어가 정확히 3개인가
- vocabularyId가 중복되지 않는가
- 모든 surfaceForm이 원문에 실제로 존재하는가
- segments를 연결한 결과가 원문과 완전히 일치하는가
- 모든 vocabularyId가 vocabulary 데이터와 연결되는가
- 읽는 법과 한국어 뜻이 비어 있지 않은가
- AI가 원문에 없는 문장을 추가하지 않았는가

검증을 통과하지 못한 결과는 저장하지 않는다.

- HTTP 응답 오류

- 순위가 중복됨
- 별자리가 중복됨
- 일본어 운세 문장이 비어 있음

- 페이지 메뉴 또는 광고 문구가 운세에 포함됨

### 크롤링 재시도
- 최초 크롤링에 실패하면 일정 시간 후 한 번 다시 시도한다.
- 자동 재시도에도 실패하면 실패 기록을 저장한다.
- 이전 날짜의 데이터를 오늘 운세처럼 표시하지 않는다.
- 저장된 최신 데이터가 있다면 실제 운세 날짜를 명확하게 표시한다.

표시 문구:
> 오늘 운세를 아직 불러오지 못했어요.
> 현재 7월 10일에 공개된 운세를 표시하고 있어요.

최신 데이터도 없으면 다음 화면을 표시한다.
> 오늘의 운세를 준비하고 있어요. 잠시 후 다시 확인해 주세요.

### AI 처리 실패
- JSON 구조가 올바르지 않으면 한 번 다시 요청한다.
- 핵심 단어가 정확히 3개가 아니면 결과를 폐기한다.
- 원문에 없는 단어가 포함되면 결과를 폐기한다.
- originalText가 원문과 다르면 결과를 폐기한다.
- segments를 연결한 문장이 원문과 다르면 결과를 폐기한다.
- 읽는 법 또는 번역 데이터가 비어 있으면 결과를 폐기한다.

AI 처리에 실패해도 크롤링된 일본어 원문은 확인할 수 있도록 한다.

표시 문구:
> 단어 학습 정보를 준비하지 못했어요. 일본어 운세 원문은 확인할 수 있어요.

### 단어 저장 실패
단어 저장 요청이 실패하면 기존 화면을 유지하며 다음 안내를 표시한다.
> 단어를 저장하지 못했어요. 네트워크 상태를 확인한 뒤 다시 시도해 주세요.

## 18. LocalStorage 및 SessionStorage

### LocalStorage 저장 항목
사용자 학습 데이터는 LocalStorage에 저장하지 않는다.
다음과 같은 비개인 설정만 저장한다.
- 마지막으로 선택한 별자리 ID
- 읽는 법 펼침 여부
- 전체 해석 펼침 여부

### SessionStorage 저장 항목
Google 로그인 화면으로 이동하기 전에 다음 정보를 일시적으로 저장한다.
- 로그인 전 보고 있던 페이지 URL
- 저장하려던 단어 ID
- 선택한 별자리 ID

로그인 완료 또는 취소 이후 임시 데이터를 삭제한다.

## 21. 인증 및 보안

### 인증 방식
- Google OAuth만 지원한다.
- Supabase Auth를 사용한다.
- 로그인 세션은 Supabase SSR 방식으로 처리한다.
- 로그인 완료 후 사용자 ID를 기준으로 저장 단어를 조회한다.

### 별도로 수집하지 않는 정보
- 생년월일
- 출생 시간
- 성별
- 전화번호
- 주소
- 일본어 학습 수준
- 사용자의 실제 별자리
- 프로필 소개

### 접근 정책
- 운세와 단어 데이터는 로그인 여부와 관계없이 조회할 수 있다.
- 저장 단어 데이터는 로그인한 사용자만 생성할 수 있다.
- 사용자는 자신의 저장 단어만 조회하고 삭제할 수 있다.
- 다른 사용자의 저장 단어에는 접근할 수 없다.
- saved_vocabulary 테이블에 RLS를 적용한다.
- Gemini API 키는 클라이언트 코드에 노출하지 않는다.
- Supabase Service Role Key는 클라이언트 코드에 노출하지 않는다.
- 크롤링 API는 인증된 크론 요청만 실행할 수 있도록 한다.
- 크롤링 API를 일반 사용자가 직접 실행할 수 없도록 한다.

## 23. 마일스톤

### M1. 더미 데이터 기반 운세 학습
- 홈에서 오늘의 12개 별자리 순위를 확인할 수 있다.
- 별자리 상세 화면으로 이동할 수 있다.
- 일본어 운세 원문이 먼저 표시된다.
- 핵심 단어 3개가 문장 안에서 강조된다.
- 단어를 누르면 읽는 법과 뜻이 표시된다.
- 원문의 읽는 법을 펼쳐볼 수 있다.
- 전체 한국어 해석을 사용자가 직접 공개할 수 있다.
- 행운의 장소 또는 아이템을 확인할 수 있다.
- 출처 영역이 표시된다.
- 모바일 화면에 맞게 동작한다.

### M2. Google 로그인과 저장 기능
- Supabase 프로젝트를 연결한다.
- Google 로그인이 동작한다.
- 로그인 상태를 확인할 수 있다.
- 비로그인 사용자가 저장 버튼을 누르면 로그인 안내가 표시된다.
- 로그인 후 기존 운세 화면으로 돌아온다.
- 로그인 전에 선택한 단어가 자동으로 저장된다.
- 사용자별로 저장 데이터가 분리된다.
- 저장과 저장 해제가 동작한다.
- 동일한 단어가 중복 저장되지 않는다.
- RLS를 적용해 자신의 저장 데이터에만 접근할 수 있다.

### M3. 저장 단어 플래시카드
- 저장한 단어 개수를 확인할 수 있다.
- 저장한 단어를 불러올 수 있다.
- 플래시카드 앞면과 뒷면을 전환할 수 있다.
- 읽는 법과 뜻을 확인할 수 있다.
- 단어가 사용된 운세 문장과 해석을 확인할 수 있다.
- 이전 카드와 다음 카드로 이동할 수 있다.
- 저장한 단어를 삭제할 수 있다.
- 저장한 단어가 없을 때 빈 화면이 표시된다.
- 다른 기기에서 같은 계정으로 로그인했을 때 저장 데이터가 유지된다.

### M3 보완 — 저장 성공 토스트의 바로 복습 액션 (PRD 개정)
- 단어 저장에 성공하면 "단어가 저장되었어요" 토스트에 "바로 복습" 액션이 함께 표시된다.
- 비로그인 사용자가 Google 로그인 후 pending 단어가 자동 저장된 경우에도 동일하게 표시된다.
- "바로 복습"을 선택하면 저장한 단어 화면으로 이동하며, 방금 저장한 단어가 첫 번째 플래시카드로 표시된다.
- 액션을 선택하지 않으면 현재 운세 상세 화면에 그대로 머문다(자동 이동 없음).
- 이미 저장된 단어(중복)라면 "이미 저장된 단어예요" 토스트만 표시하고 액션은 없다.

### M4. 오하아사 크롤링
- 평일 운세 페이지를 요청할 수 있다.
- 토요일 운세 페이지를 요청할 수 있다.
- 운세 날짜를 추출할 수 있다.
- 12개 별자리명을 추출할 수 있다.
- 1위부터 12위까지 순위를 추출할 수 있다.
- 별자리별 일본어 운세 원문을 추출할 수 있다.
- 행운의 장소 또는 아이템을 추출할 수 있다.
- 수집 결과를 Zod로 검증한다.
- 검증된 데이터만 Supabase에 저장한다.
- 원본 URL과 수집 시각을 함께 저장한다.
- 페이지 구조 변경 시 오류를 기록한다.
- 같은 날짜의 데이터가 중복 저장되지 않는다.
- 일요일에는 토요일 운세를 주말 운세로 표시한다.

### M5. AI 학습 데이터 생성 및 배포
- 크롤링된 원문으로 읽는 법을 생성한다.
- 한국어 전체 해석을 생성한다.
- 핵심 단어 3개를 생성한다.
- 단어의 기본형과 문장 속 활용형을 구분한다.
- 원문을 선택 가능한 문장 조각으로 분리한다.
- AI 결과를 JSON Schema와 Zod로 검증한다.
- 원문과 AI 결과가 같은지 비교한다.
- 검증을 통과한 데이터만 Supabase에 저장한다.
- AI 실패 시 일본어 원문만 표시할 수 있다.
- Vercel Cron으로 하루 한 번 자동 실행한다.
- 출처와 원문 링크를 화면에 표시한다.
- 오류 및 빈 데이터 상태를 적용한다.
- 모바일 반응형 웹을 Vercel에 배포한다.

## 24. 완료 기준
처음 방문한 사용자가 별도의 설명 없이 다음 과정을 완료할 수 있어야 한다.

1. 오늘의 12개 별자리 운세 순위를 확인한다.
2. 자신의 별자리를 선택한다.
3. 일본어 운세 원문을 먼저 읽는다.
4. 문장 안의 모르는 단어를 선택한다.
5. 단어의 읽는 법과 한국어 뜻을 확인한다.
6. 전체 한국어 해석을 열어 원문과 비교한다.
7. 저장하고 싶은 단어의 저장 버튼을 누른다.
8. Google 로그인을 완료한다.
9. 로그인 전에 선택했던 단어가 저장된 것을 확인한다.
10. 저장 화면에서 해당 단어를 플래시카드로 복습한다.
11. 다른 기기에서도 같은 Google 계정으로 저장한 단어를 확인한다.

운세 데이터는 다음 조건을 충족해야 한다.
- 당일 또는 주말 기준의 실제 데이터 날짜가 표시된다.
- 12개 별자리와 1위부터 12위까지의 순위가 모두 존재한다.
- 원문의 내용이 AI에 의해 변경되지 않는다.
- 각 운세에 핵심 단어가 정확히 3개 존재한다.
- 출처와 원문 페이지 링크가 표시된다.
- 데이터 수집 또는 AI 처리에 실패했을 때 잘못된 정보를 대신 표시하지 않는다.

- 순위가 1위부터 12위까지 중복 없이 존재하는가
- 원본 날짜가 화면에 올바르게 표시되는가
- 크롤링 성공률
- AI 학습 데이터 생성 성공률
- 저장 버튼 선택 후 로그인 완료율
- 로그인 완료 후 단어 저장 성공률
- 저장한 단어 화면 진입률
- 저장 성공 토스트의 "바로 복습" 액션 클릭률 (PRD 개정 추가)

- 원문 출처와 날짜에 대한 이해 여부

## 26. 에이전트 작업 규칙
- 마일스톤 단위로만 작업한다.
- 작업 시작 전에 현재 마일스톤의 구현 계획을 먼저 작성한다.
- 현재 마일스톤에 포함되지 않은 기능은 구현하지 않는다.
- Won't 목록에 있는 기능은 제안하거나 구현하지 않는다.
- 새로운 사용자 기능을 임의로 추가하지 않는다.
- 일본어 원문보다 한국어 해석을 먼저 노출하지 않는다.
- 핵심 단어는 일본어 문장 안에서 직접 선택할 수 있도록 구현한다.
- 한 운세당 핵심 단어는 정확히 3개만 제공한다.
- 비로그인 사용자에게 첫 화면부터 로그인을 요구하지 않는다.
- 단어 저장을 시도하는 시점에만 로그인 안내를 표시한다.
- 로그인 후 사용자가 보고 있던 화면으로 복귀시킨다.
- 로그인 전에 선택한 단어를 로그인 완료 후 저장한다.
- 사용자 학습 데이터는 LocalStorage에 저장하지 않는다.
- 저장 단어는 Supabase 데이터베이스에 저장한다.
- 사용자는 자신의 저장 데이터에만 접근할 수 있도록 RLS를 적용한다.
- Gemini API 키와 Supabase 서버용 키를 클라이언트에 노출하지 않는다.
- 오하아사 크롤링은 서버에서 하루 최대 한 번만 실행한다.
- 사용자 요청마다 원본 페이지를 다시 크롤링하지 않는다.
- 공개 페이지의 텍스트만 수집한다.
- 이미지, 영상, 광고 데이터는 수집하지 않는다.
- 로그인, CAPTCHA, 접근 제한을 우회하지 않는다.
- 크롤링 결과가 12개가 아니면 데이터베이스에 저장하지 않는다.
- 순위가 1위부터 12위까지 존재하지 않으면 저장하지 않는다.
- 원본 URL과 원본 날짜를 반드시 저장한다.
- 이전 날짜의 데이터를 오늘 데이터처럼 표시하지 않는다.
- 일요일에는 토요일 운세를 주말 운세로 표시한다.
- 페이지 구조가 변경되면 임의로 데이터를 추측하지 않는다.
- 크롤링한 일본어 원문을 AI가 수정하지 못하도록 검증한다.
- 사용자 접속 시 실시간으로 Gemini API를 호출하지 않는다.
- AI 결과는 JSON Schema와 Zod 검증을 통과한 경우에만 저장한다.
- 핵심 단어가 정확히 3개인지 검증한다.
- 원문에 없는 단어가 선정되면 결과를 폐기한다.
- segments를 연결한 결과가 원문과 다른 경우 폐기한다.
- 화면에 콘텐츠 출처와 비공식 서비스 안내를 표시한다.
- 마일스톤 완료 시 체크박스를 갱신한다.
- 완료한 작업, 변경한 파일, 남아 있는 오류를 함께 기록한다.

### 바로 복습 액션 관련 규칙 (PRD 개정 추가)
- 저장 성공 토스트에만 "바로 복습" 액션을 추가하며, 별도의 즉시 복습 화면·컴포넌트·LocalStorage 기록은 만들지 않는다.
- 액션은 저장 결과가 saved일 때만 표시하고, duplicate·오류에는 표시하지 않는다.
- 액션을 선택하지 않으면 자동으로 저장 탭으로 이동시키지 않는다.
- 저장 화면의 최신순 정렬을 그대로 활용하며 별도의 하이라이트 상태를 추가하지 않는다.
- 정답률, 점수, 복습 횟수 등 학습 기록은 저장하지 않는다.
