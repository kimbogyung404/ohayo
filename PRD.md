# OHAYO!

## 1. 한 줄 정의
일본 문화와 운세에 관심이 많은 20대 일본어 초급 학습자가 별도의 공부 시간을 마련해야 한다는 부담 없이, 오늘의 일본식 별자리 운세를 읽는 과정에서 자연스럽게 일본어 단어를 학습하고 저장한 단어를 반복해서 복습할 수 있도록 돕는 모바일 웹 서비스.

---

## 2. 문제와 타깃

### 문제
20대 학습자는 전공 공부, 취업 준비, 아르바이트뿐 아니라 즐길 수 있는 콘텐츠도 많아 일본어 학습에 별도의 시간과 노력을 꾸준히 투자하기 어렵다.

기존 일본어 학습 앱이나 교재는 학습을 목적으로 직접 실행해야 하며, 반복적인 문제 풀이와 진도 중심의 방식으로 인해 흥미를 잃고 며칠 만에 중단하는 경우가 많다.

반면 운세 콘텐츠는 결과에 대한 궁금증을 바탕으로 매일 짧은 시간 동안 부담 없이 반복해서 확인한다. 따라서 운세를 확인하는 행동 안에 단어 학습을 결합하면 사용자가 별도의 학습을 시작하지 않아도 자연스럽게 일본어를 접하도록 만들 수 있다.

### 타깃
일본 문화와 일본 콘텐츠를 좋아하며, 재미와 흥미를 중심으로 가볍게 시작할 수 있는 공부 방식을 선호하는 20대 일본어 초급 학습자.

- SNS에서 일본식 별자리 운세를 자주 확인한다.
- 히라가나는 읽을 수 있지만 일본어 문장은 번역기에 의존한다.
- 일본어 학습 앱을 사용해 봤지만 반복적인 학습 방식에 흥미를 잃은 경험이 있다.
- 긴 학습보다 하루에 짧게 반복할 수 있는 콘텐츠를 선호한다.
- 학습한 단어를 저장해 나중에 다시 확인하고 싶어 한다.

## 3. 핵심 시나리오 (PRD 개정: 한국어 인라인 단어 학습 흐름 반영)
운세를 확인한 다음 별도의 학습을 시작하게 하는 것이 아니라, 운세의 내용을 알아가는 과정 자체가 일본어 학습이 되도록 설계한다. 사용자는 한국어로 먼저 운세를 이해하고, 문장 속에 강조된 핵심 단어 3개만 자연스럽게 일본어로 접한다.

1. 사용자는 오늘의 12개 별자리 운세 순위를 확인한다.
2. 자신의 별자리를 선택해 상세 운세 화면으로 이동한다.
3. 상세 화면에 진입하면 한국어로 번역된 오늘의 운세 본문이 기본으로 표시된다. 문장 안의 핵심 단어 3개만 일본어로 강조 표시된다.
4. 강조된 일본어 단어를 누르면 화면 하단에 일본어 단어 카드(단어, 읽는 법, 발음 듣기)가 앞면으로 열린다. 카드는 3초 뒤 자동으로 닫히며, dim 배경을 누르거나 Escape를 눌러 즉시 닫을 수도 있다.
5. 단어를 누른 순간 확인 처리되고, 본문 안에서 그 단어가 있던 자리가 실제 문장 속에서 사용된 한국어 활용형으로 바뀐다. 같은 단어가 여러 위치에 등장해도 모든 위치가 동시에 바뀌며 확인 개수는 한 번만 늘어난다.
6. 확인한 개수(0/3~3/3)가 화면 하단 CTA에 표시된다. 3개를 모두 확인하기 전에는 "단어 확인하기 N/3"이 비활성 상태다.
7. 핵심 단어 3개를 모두 확인하면 CTA가 "확인한 단어 복습하기"로 활성화된다. 자동으로 다음 단계로 넘어가지 않으며, 사용자가 버튼을 직접 눌러야 복습 화면으로 이동한다.
8. 복습 화면에서 저장하고 싶은 단어를 직접 선택한다. 기본은 전부 미선택 상태이며, 이미 저장된 단어는 선택할 수 없는 상태로 표시된다.
9. 선택한 단어 저장하기를 누른다. 로그인하지 않은 사용자는 Google 로그인 안내를 확인한다.
10. Google 로그인을 완료하면 같은 별자리의 복습 화면으로 돌아오며, 선택했던 단어가 그대로 복원된다. 자동 저장되지 않으며 사용자가 저장하기를 다시 눌러야 저장이 진행된다.
11. 저장이 완료되면 완료 화면이 표시된다. 완료 화면에서 운세로 돌아가거나 저장된 단어 화면으로 이동할 수 있다.
12. 이후 저장한 단어 화면에서 저장한 단어를 반복해서 복습한다.
13. 같은 Google 계정으로 로그인하면 다른 기기에서도 저장한 단어를 확인할 수 있다.

## 4. 서비스 및 콘텐츠 원칙

### 학습 원칙 (PRD 개정: 한국어 인라인 단어 학습 흐름 반영)
- 한국어로 번역된 운세 본문을 상세 화면 진입 시 기본으로 노출한다. 별도의 "한국어 해석 보기/가리기" 단계는 두지 않는다.
- 문장 속 핵심 단어 3개만 확인 전에는 일본어로 표시하고, 확인 후에는 실제 문장에서 쓰인 한국어 활용형으로 그 자리에서 바로 전환한다.
- 사용자가 강조된 단어를 직접 눌러 확인하도록 한다.
- 한 운세당 학습 단어는 정확히 3개만 제공한다.
- 긴 설명보다 읽는 법, 뜻, 문장 속 사용 예시를 중심으로 제공한다.
- 운세 확인과 단어 학습이 하나의 화면 안에서 이어지도록 한다.
- 한국어 문장과 일본어 원문을 프론트에서 임의로 매칭하지 않는다. 어떤 위치가 어떤 단어인지는 M5가 생성해 DB에 저장한 값을 그대로 사용한다.

### 콘텐츠 원칙
- 오하아사 운세 페이지에 공개된 당일 별자리 순위와 일본어 운세 문장을 수집한다.
- 운세 순위와 원문의 내용은 AI가 새로 만들거나 수정하지 않는다.
- AI는 읽는 법, 한국어 번역, 핵심 단어 데이터 생성에만 사용한다.
- 화면에 운세의 원본 날짜와 출처를 표시한다.
- 오하아사 또는 ABC TV에서 직접 운영하는 공식 서비스처럼 표현하지 않는다.
- 이미지, 영상, 광고 콘텐츠는 수집하지 않는다.
- 로그인, CAPTCHA, 접근 제한이 있는 페이지를 우회하지 않는다.

### 출처 안내 문구 (PRD 개정: 노출 위치를 홈 화면으로 정정)
다음 내용을 홈 화면 하단에 표시한다(상세 운세 화면에는 별도로 반복 표시하지 않는다 — 7절 화면 2 참고).

> 운세 출처: ABC TV 「おはよう朝日です」
> OHAYO!는 ABC TV의 공식 서비스가 아닙니다.

## 5. 기능 범위

### Must · MVP 1: 운세를 읽으며 단어 학습 (PRD 개정: 한국어 인라인 단어 학습 흐름으로 전면 개편)
- 오늘 날짜 표시
- 오늘의 12개 별자리 운세 순위 확인
- 별자리별 상세 운세 제공
- 한국어로 번역된 운세 본문을 상세 화면 진입 시 기본 노출 (PRD 개정)
- 문장 속 핵심 단어 3개만 확인 전 일본어로 강조 표시, 나머지 본문은 항상 한국어 (PRD 개정)
- 행운의 장소·아이템도 한국어 본문 기준으로 동일하게 처리 (PRD 개정)
- 강조된 단어를 누르면 화면 하단에 일본어 단어 카드(단어, 읽는 법, 발음 듣기)가 앞면으로 열리며 확인 처리 (PRD 개정)
- 단어 카드는 뒤집기 없이 앞면만 표시, 3초 뒤 자동 종료, dim 배경 터치 또는 Escape로 즉시 종료 (PRD 개정)
- 확인한 단어가 있던 자리는 실제 문장 속 한국어 활용형으로 즉시 전환, 같은 단어의 모든 위치가 동시에 전환 (PRD 개정)
- 확인 상태와 카운트는 `vocabulary.id` 기준으로 관리, 같은 단어를 여러 위치에서 눌러도 확인 개수는 한 번만 증가 (PRD 개정)
- 확인한 개수(0/3~3/3)를 하단 CTA에 표시(`단어 확인하기 N/3` → 3/3에서 `확인한 단어 복습하기`로 전환) (PRD 개정)
- 3/3을 달성해도 자동으로 다음 단계로 이동하지 않으며, CTA를 직접 눌러야 복습 화면으로 이동 (PRD 개정)
- study·review·complete 세 단계 모두 화면 하단 고정 액션 바(StickyActionBar)를 공용으로 사용 (PRD 개정)
- 복습 화면에서 저장할 단어 개별 선택, 저장 및 저장 해제
- 별도의 "한국어 해석 보기/가리기" 단계와 번역 박스는 두지 않는다(PRD 개정: 폐기)
- 출처와 비공식 서비스 안내는 상세 화면에 노출하지 않는다(PRD 개정: 홈 화면으로 이동, 4절 참고). 원문 페이지 이동 링크는 상세 화면에서 삭제됐고 아직 대체 위치가 없다(PRD 개정: 확인된 격차, 24절 참고)

### Must · MVP 2: Google 로그인과 사용자별 저장
- Google OAuth 로그인
- 로그인 상태 확인
- 로그아웃
- 비로그인 상태에서 저장 버튼 선택 시 로그인 안내
- 로그인 완료 후 같은 별자리의 복습 화면으로 복귀
- 로그인 전에 선택했던 단어 목록 복원 (자동 저장은 하지 않으며, 사용자가 저장하기를 다시 눌러야 진행됨) (PRD 개정)
- 사용자 계정별 단어 저장(여러 단어 한 번에 저장)
- 동일 단어 중복 저장 방지
- 다른 기기에서 저장 데이터 동기화

### Must · MVP 3: 저장 단어 플래시카드 복습
- 저장한 단어 개수 확인
- 저장한 단어를 플래시카드 형식으로 확인
- 카드 앞면에 일본어 단어 표시
- 카드를 누르면 뒷면으로 전환
- 카드 뒷면에 읽는 법과 한국어 뜻 표시
- 단어가 사용된 일본어 운세 문장 표시
- 운세 문장의 한국어 해석 표시
- 단어 저장 날짜 표시
- 이전 카드와 다음 카드 이동
- 저장한 단어 삭제
- 저장한 단어가 없을 때 빈 화면 표시

### Must · M3 보완: 저장 성공 토스트의 바로 복습 액션 (PRD 개정, 상세 화면 학습 플로우 개편으로 대체됨)
- (기존) 단어 저장 성공 시 성공 토스트에 "바로 복습" 액션 표시
- (기존) "바로 복습" 선택 시 저장한 단어 화면으로 이동, 방금 저장한 단어부터 플래시카드로 복습
- 상세 화면의 단어 저장은 이제 토스트가 아니라 전용 완료 화면으로 안내한다. 완료 화면에서 운세로 돌아가기 또는 저장된 단어 보기를 선택할 수 있다. (PRD 개정)
- 이미 저장된 단어(중복)는 복습 화면의 선택 대상에서 처음부터 제외한다. (PRD 개정)

### Must · MVP 4: 오하아사 운세 수집
- 평일 운세 페이지 데이터 수집
- 토요일 운세 페이지 데이터 수집
- 운세 날짜 추출
- 12개 별자리 순위 추출
- 별자리별 일본어 운세 원문 추출
- 행운의 장소 또는 아이템 추출
- 수집 데이터 검증
- 검증된 데이터만 데이터베이스에 저장
- 수집 시각과 원본 URL 저장
- 하루 한 번 예약 실행
- 수집 실패 기록

### Must · MVP 5: 한국어 인라인 학습 데이터 생성 (M5) (PRD 개정 추가: 구현 완료)
- M5가 한국어 번역문(`koreanTranslation`) 안에서 각 핵심 단어가 실제로 쓰인 한국어 활용형(`koreanText`)을 생성
- M5가 행운의 장소·아이템의 한국어 번역(`luckyItemKo`)도 함께 생성(기존에는 미생성 상태였음)
- `koreanText`가 `koreanTranslation` 또는 `luckyItemKo` 중 최소 한 곳에 정확히 한 번만 등장하는지 검증(0회·중복 등장은 실패 처리)
- 검증을 통과한 위치 정보로 `korean_segments`/`lucky_item_ko_segments`를 서버에서 결정론적으로 조립해 저장(일본어 `segments`와 동일한 방식 — AI가 세그먼트 배열 자체를 만들지 않는다)
- 프론트는 저장된 세그먼트를 그대로 렌더링만 하며, 문자열 매칭이나 형태소 추측을 하지 않는다
- 기존에 AI 처리까지 완료(`ai_status='success'`)된 24개 운세(2026-07-11, 2026-07-13)에 대해 백필을 완료해 전부 새 필드를 채움

### Won't
- 출석 체크
- 운세 공유
- 학습 난이도 설정
- 기억했어요, 아직 헷갈려요 등의 복습 상태 기록
- 단어 검색
- 단어 카테고리 및 태그 분류
- 저장된 단어를 활용한 퀴즈
- 정답률 기록
- 단어별 메모 작성
- 핵심 단어 외 표현 직접 추가
- 문법 설명
- 사용자 프로필 편집
- 이메일과 비밀번호 회원가입
- 생년월일을 통한 별자리 자동 설정
- 출생 시간 입력
- 사용자 접속 시 실시간 운세 생성
- 별도의 관리자 페이지
- 이미지 및 영상 크롤링
- 오하아사 페이지의 접근 제한 우회
- 로그인 없는 즉시 복습(인라인 리뷰 카드), 최근 본 단어 LocalStorage 저장, 홈 "지난번 본 단어" 카드 (MVP 범위 제외, PRD 개정)

## 6. 로그인 정책

### 비로그인 사용자가 할 수 있는 행동
- 오늘의 별자리 순위 확인
- 별자리별 상세 운세 확인(한국어 본문은 로그인 여부와 무관하게 기본으로 노출됨) (PRD 개정)
- 핵심 단어를 눌러 읽는 법 확인, 확인 후 한국어 활용형으로 전환 (PRD 개정)
- 행운의 장소 또는 아이템 확인

### 로그인이 필요한 행동
- 단어 저장
- 저장한 단어 확인
- 저장한 단어 삭제
- 다른 기기에서 저장 데이터 불러오기

### 로그인 노출 시점
서비스에 처음 접속했을 때는 로그인을 요구하지 않는다.
사용자가 단어를 저장하려는 순간 다음 내용의 로그인 안내를 표시한다.

> 단어를 저장하려면 로그인이 필요해요.
> Google 계정으로 로그인하면 저장한 단어를 다시 복습하고 다른 기기에서도 이어서 학습할 수 있어요.

### 로그인 완료 후 처리 (PRD 개정: 자동 저장 → 선택 상태 복원 후 사용자가 이어서 진행)
- 로그인 전에 보고 있던 별자리 ID를 임시로 저장한다.
- 로그인 전에 복습 화면에서 선택했던 단어 ID 목록을 임시로 저장한다.
- 로그인 완료 후 같은 별자리의 복습 화면으로 돌아오며, 선택했던 단어 목록을 그대로 복원한다.
- 로그인 전에 선택했던 단어를 자동으로 저장하지 않는다. 사용자가 저장하기를 다시 눌러야 저장이 진행된다.
- 이미 저장된 단어라면 중복 저장하지 않는다.
- 복원 후(또는 저장 성공 후) 임시 데이터를 삭제한다.
- 로그인 취소 시 기존 운세 화면을 유지한다.

## 7. 화면 구성

### 화면 1. 홈
오늘의 12개 별자리 운세 순위를 확인하는 화면.

구성 요소
- 서비스 로고
- 오늘 날짜
- 오늘의 운세 안내 문구
- 운세 데이터 기준 날짜

- 별자리 아이콘
- 일본어 별자리명
- 한국어 별자리명

- 운세 출처 안내
- 하단 내비게이션
  - 오늘의 운세
  - 저장한 단어

### 화면 2. 별자리 상세 운세 (PRD 개정: 한국어 인라인 단어 학습 흐름으로 전면 개편)
선택한 별자리의 운세를 한국어로 읽으면서 문장 속 핵심 단어 3개만 일본어로 접하고, 그 자리에서 바로 복습하고 저장까지 이어지는 화면.
하나의 라우트 안에서 `study(학습) → review(복습) → complete(완료)` 세 단계로 구성된다. 세 단계 모두 화면 하단 고정 액션 바(StickyActionBar)를 공용으로 사용한다.

#### study 단계
구성 요소
- 뒤로 가기, `{순위}위 {별자리 한국어명}` 제목
- `1. 오늘의 운세`: 한국어 운세 본문(`korean_segments` 기준). 일반 문장은 한국어, 핵심 단어 3개만 확인 전 일본어로 강조 표시
- `2. 행운의 장소와 아이템`: 동일한 방식(`lucky_item_ko_segments` 기준)
- 강조된 단어를 누르면 화면 하단에 단어 카드가 오버레이로 열린다. 카드는 뒤집기 없이 앞면만 표시하며 일본어 단어, 읽는 법, 발음 듣기를 보여준다(뜻은 표시하지 않는다)
- 카드는 3초 뒤 자동으로 닫힌다. dim 배경을 누르거나 Escape를 누르면 즉시 닫히고, 카드 내부를 눌러도 닫히지 않는다. 같은 단어를 다시 누르면 3초 타이머가 처음부터 다시 시작된다
- 단어를 누른 순간 확인 처리되며(`vocabulary.id` 기준), 본문 안에서 해당 단어가 있던 자리가 실제 문장 속 한국어 활용형으로 즉시 전환된다. 같은 단어가 여러 위치에 등장해도 모든 위치가 동시에 전환되고 확인 개수는 한 번만 증가한다
- 화면 하단 StickyActionBar에 확인 진행 CTA를 고정 표시한다
  - 0~2개 확인: `단어 확인하기 N/3`, 비활성 상태, 그 위에 "강조된 일본어 단어 3개를 눌러 뜻을 확인해보세요" 툴팁 표시
  - 3개 모두 확인: `확인한 단어 복습하기`, 활성 상태. 자동으로 다음 단계로 넘어가지 않으며 사용자가 눌러야 review로 이동

데이터 안내: 행운의 장소·아이템의 한국어 데이터(`lucky_item_ko`, `lucky_item_ko_segments`)는 M5가 생성해 DB에 저장한다(20절 데이터베이스 구조 참고). 프론트는 이 값을 그대로 렌더링만 하며 문자열을 임의로 매칭하지 않는다. 두 섹션 모두 실제 데이터가 없으면(백필 전이거나 생성 실패) study 단계 대신 "한국어 학습 데이터를 준비하고 있어요" 안내와 홈으로 돌아가기 버튼만 표시한다.

운세 출처와 비공식 서비스 안내는 상세 화면에 표시하지 않는다(4절 참고 — 홈 화면 하단으로 이동). 원문 페이지 이동 링크는 상세 화면 개편으로 삭제됐고 아직 대체 위치가 없다(24절 참고).

#### review 단계
- 제목: `단어 복습하기`
- 상단 안내 문구: `오늘의 단어를 복습해보세요`
- 보조 문구: "방금 공부한 단어들을 단어장에 저장할까요?"
- 방금 확인한 핵심 단어 3개를 카드로 나열, 기본은 전부 미선택 상태이며 사용자가 저장할 단어를 개별 선택한다(이미 저장된 단어는 선택할 수 없는 상태로 표시하고 선택 개수에도 포함하지 않음)
- 각 카드에 일본어 단어, 읽는 법, 한국어 뜻을 표시하고 발음 듣기는 카드 우측에 정렬
- 카드 다중 선택/해제
- StickyActionBar에 종료하기(학습 단계로 복귀) / 선택 개수에 따른 저장 버튼(`{N}개 저장하기`, 0개면 비활성)을 고정 표시
- 비로그인 상태에서 저장을 누르면 기존 Google 로그인 안내를 표시하고, 로그인 완료 후 같은 화면·선택 상태로 돌아온다

#### complete 단계
- 완료 아이콘과 "단어 저장이 완료됐어요!" 안내(위치 유지)
- StickyActionBar에 운세로 돌아가기(학습 단계로 복귀) / 저장된 단어 보기(저장한 단어 화면으로 이동)를 고정 표시

정보 노출 순서 (PRD 개정)
1. 한국어로 번역된 오늘의 운세와 행운의 장소·아이템(상세 화면 진입 시 기본 노출)
2. 문장 속 핵심 단어 3개만 확인 전 일본어로 강조
3. 사용자가 단어를 누를 때마다 해당 위치가 한국어 활용형으로 전환
4. 핵심 단어 3개를 모두 확인한 이후 복습·저장으로 진행

한국어 본문은 확인 여부와 무관하게 항상 먼저 노출되며, 더 이상 "핵심 단어를 모두 확인해야 해석이 열리는" 게이팅은 존재하지 않는다.

### 화면 3. 로그인 안내

진입 경로 (PRD 개정 추가): 저장 성공 토스트의 "바로 복습" 액션을 선택하면 이 화면으로 이동하며, 저장한 단어는 저장 시각 최신순으로 정렬되어 있어 방금 저장한 단어가 첫 번째 플래시카드로 표시된다.

- 하단 내비게이션

## 8. 운세 데이터 수집 방식

### 전체 데이터 흐름
OHAYO!는 오하아사 공개 JSON 엔드포인트에 게시된 당일 운세를 서버에서 하루 한 번 수집한다(M4).

수집한 일본어 운세 원문은 이후 별도 단계(M5)에서 Gemini API를 통해 일본어 읽는 법, 자연스러운 한국어 번역, 초급 학습자를 위한 핵심 단어 3개로 가공한다.

사용자가 서비스에 접속할 때마다 오하아사 엔드포인트나 Gemini API를 호출하지 않고, 사전에 Supabase에 저장된 당일 데이터를 불러온다.

**M4와 M5의 역할 구분**
- **M4(운세 수집)**: 원문·행운 아이템·날짜·별자리·순위만 수집해 `fortunes`에 저장한다. `ai_status='pending'`으로 남기며, `reading_text`/`korean_translation`/`segments`/`vocabulary`는 만들지 않는다.
- **M5(학습 데이터 생성, PRD 개정: 구현 완료)**: M4가 저장한 원문과 행운 아이템을 입력으로 Gemini API를 호출해 읽는 법·한국어 번역·핵심 단어·핵심 단어별 한국어 활용형(`koreanText`)·행운 아이템 한국어 번역(`luckyItemKo`)을 생성한다. 검증을 통과한 위치 정보로 `segments`/`korean_segments`/`lucky_item_ko_segments`를 서버에서 결정론적으로 조립해 저장한다. M4는 M5의 입력 데이터만 준비할 뿐, AI 처리는 하지 않는다.

### 데이터 처리 순서 (M4)
1. 서버에서 오하아사 공개 JSON 엔드포인트를 하루 한 번 요청한다(HTML 파싱 없음, 아래 11번 참고).
2. 응답의 `onair_date` 필드로 운세 날짜를 확인한다(URL로 날짜를 계산하지 않는다).
3. 12개 별자리의 순위와 일본어 운세 원문을 응답에서 그대로 추출한다.
4. 행운의 장소 또는 아이템을 함께 추출한다.
5. 수집 데이터가 올바른 형식인지 검증한다(12번 섹션).
6. 검증된 원본 데이터만 Supabase에 저장한다(`ai_status='pending'`).

### 데이터 처리 순서 (M5, PRD 개정: 구현 완료)
7. Gemini API에 M4가 저장한 일본어 운세 원문과 행운의 장소·아이템을 전달한다.
8. 읽는 법, 한국어 번역, 행운 아이템 한국어 번역(`luckyItemKo`), 핵심 단어 3개(단어별 한국어 활용형 `koreanText` 포함)를 생성한다.
9. AI 응답의 JSON 구조, 단어 개수, `koreanText`가 한국어 번역 또는 행운 아이템 번역 중 최소 한 곳에 정확히 한 번 등장하는지를 검증한다.
10. 검증을 통과한 위치 정보로 `segments`/`korean_segments`/`lucky_item_ko_segments`를 결정론적으로 조립하고, 원문과 글자 단위로 일치하는지 재확인한 뒤 Supabase에 저장한다.
11. 사용자는 데이터베이스에 저장된 운세를 조회한다.
12. 이미 `ai_status='success'`로 저장된 기존 운세(2026-07-11, 2026-07-13, 총 24개)는 별도 백필 스크립트로 `korean_segments`/`lucky_item_ko`/`lucky_item_ko_segments`만 채웠다. `original_text`/`korean_translation`/`lucky_item`/`vocabulary`는 백필 과정에서 재생성하거나 수정하지 않았다.

## 9. 크롤링 대상 및 요일 처리

### 실제 조사 결과 (PRD 개정)
오하아사 평일 페이지(`/ohaasa/week/horoscope/`)와 토요일 페이지(`/ohaasa/sat/horoscope/`)는 화면(HTML)만 다를 뿐, 실제 운세 데이터는 **평일·토요일 구분 없이 동일한 공개 JSON 엔드포인트** 하나에서 제공된다.

```
https://www.asahi.co.jp/data/ohaasa2020/horoscope.json
```

두 페이지 모두 이 JSON을 그대로 가져와 화면에 표시한다. 따라서 서버는 이 엔드포인트 하나만 요청하면 되고, 요일에 따라 다른 URL을 요청할 필요가 없다.

### 평일 운세
- 대상 요일: 월요일~금요일
- 데이터: 응답의 `onair_date`가 해당 평일인 오늘의 별자리 순위와 원문

### 토요일 운세
- 대상 요일: 토요일
- 데이터: 응답의 `onair_date`가 토요일인 별자리 순위와 원문

### 일요일 처리
오하아사에서 일요일 운세가 별도로 제공되지 않으므로, 일요일에 같은 엔드포인트를 요청하면 `onair_date`가 토요일 날짜 그대로 반환된다(응답 자체가 갱신되지 않음). 이 경우 토요일에 수집한 데이터를 주말 운세로 일요일까지 유지한다.

일요일에는 오늘 새롭게 공개된 운세처럼 표현하지 않고 다음 안내를 표시한다.

> 토요일에 공개된 이번 주말 운세예요.

### 시간대 기준
- 서비스 날짜 기준: Asia/Seoul
- 원본 페이지 날짜 확인 기준: Asia/Tokyo
- 한국과 일본의 시차는 없지만 코드에서는 시간대를 명시적으로 지정한다.

## 10. 크롤링 정책
- 크롤링은 사용자 요청마다 실행하지 않는다.
- 서버 예약 작업을 통해 하루 최대 한 번 실행한다.
- 최초 실행에 실패했을 때만 한 번 재시도한다.
- 로그인 없이 접근 가능한 공개 페이지의 텍스트만 수집한다.
- CAPTCHA, 로그인, 접근 제한 또는 차단을 우회하지 않는다.
- 이미지, 영상, 광고, 사용자 데이터는 수집하지 않는다.
- 운세 학습에 필요한 텍스트만 수집한다.
- 원본 HTML 전체를 사용자 화면에 노출하지 않는다.
- 수집 데이터에 원본 페이지 URL과 수집 시각을 함께 기록한다.
- 페이지의 운세 날짜와 실제 서비스 날짜를 비교한다.
- 페이지 구조가 변경되어 12개 별자리를 모두 찾지 못하면 저장하지 않는다.
- 기존에 저장된 같은 날짜의 데이터가 있으면 중복 저장하지 않는다.
- 이전 날짜의 데이터를 오늘 데이터처럼 표시하지 않는다.

## 12. 크롤링 검증
수집 결과가 다음 조건을 모두 충족해야 정상 데이터로 처리한다(M4 구현 기준).

- 응답이 스키마(배열, `onair_date`, `detail` 배열 등)와 일치하는가
- `onair_date`가 실존하는 날짜인가(예: 20260231처럼 존재하지 않는 날짜 거부)

- 같은 별자리가 중복되지 않았는가
- 순위(`ranking_no`)가 1위부터 12위까지 존재하는가
- 같은 순위가 중복되지 않았는가
- 일본어 운세 원문이 비어 있지 않은가
- 행운의 아이템이 비어 있지 않은가

하나라도 실패하면 전체를 실패로 처리하며, 검증에 실패한 결과는 `fortunes`에 부분 저장하지 않는다.

## 13. AI 동작 정의
AI는 크롤링이 완료되고 검증된 운세 원문에만 사용한다.
AI는 운세의 순위나 내용을 새로 생성하지 않는다.

### 입력
- 날짜
- 별자리 ID
- 일본어 별자리명
- 한국어 별자리명
- 별자리 순위
- 일본어 운세 원문
- 행운의 장소 또는 아이템
- 원본 페이지 URL

### AI 역할
- 일본어 운세 원문의 읽는 법 생성
- 자연스러운 한국어 번역 생성
- 행운의 장소·아이템의 한국어 번역 생성(`luckyItemKo`) (PRD 개정 추가)
- 원문 안에서 핵심 단어 3개 선정
- 단어의 기본형 생성
- 문장 속 활용형 추출
- 단어의 읽는 법 생성
- 문장 안에서 사용된 의미에 맞는 한국어 뜻 생성
- 각 핵심 단어가 한국어 번역문 안에서 실제로 쓰인 활용형(`koreanText`) 식별 — 사전형/의역이 아니라 번역문에 그대로 등장하는 문자열이어야 한다 (PRD 개정 추가)
- 일본어 원문을 선택 가능한 문장 조각으로 분리

세그먼트 배열(`segments`/`korean_segments`/`lucky_item_ko_segments`) 자체는 AI가 직접 만들지 않는다. AI는 위치를 특정할 수 있는 문자열(`surfaceForm`, `koreanText`, `luckyItemKo`)만 생성하고, 서버가 그 문자열이 원문/번역문 안에 정확히 한 번 등장하는지 확인한 뒤 결정론적으로 세그먼트를 조립한다.

## 14. AI 프롬프트 규칙
- 한국인 일본어 초급 학습자를 돕는 친절한 일본어 교사 역할을 부여한다.
- 입력된 운세 원문의 내용을 변경하지 않는다.
- 새로운 운세 내용이나 문장을 추가하지 않는다.
- originalText는 입력받은 일본어 원문과 완전히 동일해야 한다.
- 원문 안에서 학습 가치가 높은 JLPT N5~N4 수준의 단어를 정확히 3개 선정한다.
- N5~N4 수준의 단어가 부족하면 초급 학습자가 알아두기 좋은 표현을 선정한다.
- 원문에 실제로 존재하는 단어만 선정한다.
- 조사, 접속사, 숫자, 단순 기호, 고유명사는 제외한다.
- 동사와 형용사의 word 값은 사전형으로 제공한다.
- surfaceForm에는 원문에 실제로 등장한 형태를 제공한다.
- 단어의 뜻은 해당 문장 안에서 사용된 의미를 기준으로 작성한다.
- 번역은 자연스럽게 작성하되 원문의 의미를 과장하거나 축소하지 않는다.
- 운세의 확정적인 표현을 임의로 강화하지 않는다.
- segments의 모든 text를 순서대로 연결했을 때 originalText와 정확히 일치해야 한다.
- 같은 문장 위치를 두 개 이상의 단어가 중복해서 사용하지 않도록 한다.
- luckyItemKo는 행운의 장소·아이템(원문)을 자연스러운 한국어로 번역한 것이어야 한다 (PRD 개정 추가).
- koreanText는 koreanTranslation 또는 luckyItemKo 안에 실제로 등장하는 문자열이어야 한다. meaning처럼 사전형이나 의역으로 바꾸지 않고, 번역문에 쓰인 활용형 그대로 반환한다 (PRD 개정 추가).
- koreanTranslation과 luckyItemKo에 없는 문자열을 koreanText로 만들지 않는다 (PRD 개정 추가).
- 지정된 JSON 구조 이외의 설명을 출력하지 않는다.

## 15. AI 결과 검증
AI 응답을 데이터베이스에 저장하기 전에 다음 조건을 검증한다.

- JSON Schema와 일치하는가
- 필수 필드가 모두 존재하는가
- 입력한 날짜와 출력 날짜가 같은가
- 입력한 별자리 ID와 출력 별자리 ID가 같은가
- 입력한 순위와 출력 순위가 같은가
- originalText가 크롤링한 원문과 완전히 같은가
- 핵심 단어가 정확히 3개인가
- vocabularyId가 중복되지 않는가
- 모든 surfaceForm이 원문에 실제로 존재하는가
- segments를 연결한 결과가 원문과 완전히 일치하는가
- 모든 vocabularyId가 vocabulary 데이터와 연결되는가
- 읽는 법과 한국어 뜻이 비어 있지 않은가
- AI가 원문에 없는 문장을 추가하지 않았는가
- luckyItemKo가 비어 있지 않은가 (PRD 개정 추가)
- 각 koreanText가 koreanTranslation 또는 luckyItemKo 중 정확히 한 곳에 한 번만 등장하는가(0회·중복 등장은 실패) (PRD 개정 추가)
- koreanText로 인해 koreanTranslation 또는 luckyItemKo 안에서 위치가 겹치는 단어가 없는가 (PRD 개정 추가)
- koreanSegments/luckyItemKoSegments를 연결한 결과가 각각 koreanTranslation/luckyItemKo와 완전히 일치하는가 (PRD 개정 추가)
- 핵심 단어 3개가 koreanSegments·luckyItemKoSegments 중 최소 한 곳에는 전부 연결됐는가 (PRD 개정 추가)

검증을 통과하지 못한 결과는 저장하지 않는다.

## 16. 실패 처리

### 크롤링 실패
다음 상황에서는 수집 결과를 저장하지 않는다.
- 원본 페이지 요청 실패
- HTTP 응답 오류
- 페이지에서 날짜를 찾지 못함
- 별자리 데이터를 12개 모두 찾지 못함
- 순위가 중복됨
- 별자리가 중복됨
- 일본어 운세 문장이 비어 있음
- 페이지 HTML 구조 변경
- 수집한 날짜가 기대 날짜와 다름
- 페이지 메뉴 또는 광고 문구가 운세에 포함됨

### 크롤링 재시도
- 최초 크롤링에 실패하면 일정 시간 후 한 번 다시 시도한다.
- 자동 재시도에도 실패하면 실패 기록을 저장한다.
- 이전 날짜의 데이터를 오늘 운세처럼 표시하지 않는다.
- 저장된 최신 데이터가 있다면 실제 운세 날짜를 명확하게 표시한다.

표시 문구:
> 오늘 운세를 아직 불러오지 못했어요.
> 현재 7월 10일에 공개된 운세를 표시하고 있어요.

최신 데이터도 없으면 다음 화면을 표시한다.
> 오늘의 운세를 준비하고 있어요. 잠시 후 다시 확인해 주세요.

### AI 처리 실패
- JSON 구조가 올바르지 않으면 한 번 다시 요청한다.
- 핵심 단어가 정확히 3개가 아니면 결과를 폐기한다.
- 원문에 없는 단어가 포함되면 결과를 폐기한다.
- originalText가 원문과 다르면 결과를 폐기한다.
- segments를 연결한 문장이 원문과 다르면 결과를 폐기한다.
- 읽는 법 또는 번역 데이터가 비어 있으면 결과를 폐기한다.
- luckyItemKo가 비어 있거나 koreanText 검증에 실패하면 결과를 폐기한다 (PRD 개정 추가).

AI 처리에 실패해도 크롤링된 일본어 원문은 확인할 수 있도록 한다.

표시 문구:
> 단어 학습 정보를 준비하지 못했어요. 일본어 운세 원문은 확인할 수 있어요.

`korean_segments`/`lucky_item_ko_segments`/`lucky_item_ko`가 없는 운세(백필 전이거나 생성 실패)는 상세 화면에서 상호작용 없는 준비 중 안내만 표시한다(PRD 개정 추가). 프론트는 이 경우에도 문자열 매칭으로 임의 대체하지 않는다.

표시 문구:
> 한국어 학습 데이터를 준비하고 있어요.

## 21. 인증 및 보안

### 인증 방식
- Google OAuth만 지원한다.
- Supabase Auth를 사용한다.
- 로그인 세션은 Supabase SSR 방식으로 처리한다.
- 로그인 완료 후 사용자 ID를 기준으로 저장 단어를 조회한다.

### 별도로 수집하지 않는 정보
- 생년월일
- 출생 시간
- 성별
- 전화번호
- 주소
- 일본어 학습 수준
- 사용자의 실제 별자리
- 프로필 소개

### 접근 정책
- 운세와 단어 데이터는 로그인 여부와 관계없이 조회할 수 있다.
- 저장 단어 데이터는 로그인한 사용자만 생성할 수 있다.
- 사용자는 자신의 저장 단어만 조회하고 삭제할 수 있다.
- 다른 사용자의 저장 단어에는 접근할 수 없다.
- saved_vocabulary 테이블에 RLS를 적용한다.
- Gemini API 키는 클라이언트 코드에 노출하지 않는다.
- Supabase Service Role Key는 클라이언트 코드에 노출하지 않는다.
- 크롤링 API는 인증된 크론 요청만 실행할 수 있도록 한다.
- 크롤링 API를 일반 사용자가 직접 실행할 수 없도록 한다.

- AI 처리 대기

### 중복 저장
이미 저장된 단어라면 새로운 데이터를 추가하지 않고 저장된 상태를 유지한다.

### 상세 화면 학습 진행 상태 (PRD 개정: 한국어 인라인 단어 학습 흐름 반영)
- 단계: `study`(학습) → `review`(복습) → `complete`(완료). 하나의 라우트 안에서 상태로만 전환하며 페이지를 이동하지 않는다. `study` 단계 안에 별도의 "해석 보기 전/후" 하위 상태는 없다 — 한국어 본문은 항상 노출된다.
- 확인한 핵심 단어: `vocabulary.id` 집합으로 관리한다. 일본어 문자열을 키로 사용하지 않으며, 같은 단어가 본문과 행운의 장소·아이템에 반복 등장해도 하나의 ID로 취급해 중복 증가하지 않는다. 이 집합은 `KoreanSegmentedText`가 각 위치를 일본어(미확인)/한국어(확인)로 렌더링하는 기준이기도 하다.
- 단어 카드 오버레이 열림 상태: 열려 있는 단어의 `vocabulary.id`(또는 null)로 관리한다. 같은 단어를 다시 눌러도 3초 자동 종료 타이머가 처음부터 재시작되도록, 클릭할 때마다 증가하는 카운터를 함께 관리한다.
- 복습 화면 선택 상태: 저장 대상으로 선택한 `vocabulary.id` 집합으로 관리하며, `review` 진입 시 항상 빈 집합(전부 미선택)에서 새로 시작한다. 이미 저장된 단어는 이 집합에 포함하지 않는다.
- `study` → `review` 전환 조건: 핵심 단어 3개를 모두 확인했을 때만 CTA가 활성화되며, 사용자가 직접 눌러야 전환한다(3/3 달성만으로 자동 전환하지 않는다). `종료하기`로 `study`에 돌아갔다가 다시 `review`에 들어가도 새 복습 세션으로 보고 선택 상태를 초기화한다.
- `review` → `complete` 전환 조건: 새로 저장된 단어가 1개 이상일 때만 전환한다. 선택한 단어가 전부 이미 저장된 상태였다면(중복만 있는 경우) "이미 저장된 단어예요" 안내만 표시하고 `review`를 유지하며, 저장 요청 자체가 실패해도 `review`를 유지한다.

### M1. 더미 데이터 기반 운세 학습 (PRD 개정: 한국어 인라인 단어 학습 흐름 반영)
- 홈에서 오늘의 12개 별자리 순위를 확인할 수 있다.

- 문장 속 핵심 단어 3개만 확인 전 일본어로 강조된다.

- 확인한 단어는 본문 안에서 실제 문장 속 한국어 활용형으로 전환된다.
- 확인 개수(0/3~3/3)가 하단 CTA에 표시되고, 3/3에서 "확인한 단어 복습하기"로 활성화된다.
- 행운의 장소 또는 아이템을 한국어로 확인할 수 있다.
- 출처 영역은 상세 화면이 아닌 홈 화면에 표시된다.
- 모바일 화면에 맞게 동작한다.

- Supabase 프로젝트를 연결한다.

### M3. 저장 단어 플래시카드
- 저장한 단어 개수를 확인할 수 있다.

- 다른 기기에서 같은 계정으로 로그인했을 때 저장 데이터가 유지된다.

### M3 보완 — 저장 성공 토스트의 바로 복습 액션 (PRD 개정)
- 단어 저장에 성공하면 "단어가 저장되었어요" 토스트에 "바로 복습" 액션이 함께 표시된다.
- 비로그인 사용자가 Google 로그인 후 pending 단어가 자동 저장된 경우에도 동일하게 표시된다.
- "바로 복습"을 선택하면 저장한 단어 화면으로 이동하며, 방금 저장한 단어가 첫 번째 플래시카드로 표시된다.
- 액션을 선택하지 않으면 현재 운세 상세 화면에 그대로 머문다(자동 이동 없음).
- 이미 저장된 단어(중복)라면 "이미 저장된 단어예요" 토스트만 표시하고 액션은 없다.

### M4. 오하아사 크롤링 (PRD 개정 — 실제 구현 기준)
- 공개 JSON 엔드포인트(`/data/ohaasa2020/horoscope.json`)를 요청할 수 있다(평일·토요일 동일 엔드포인트).

- 12개 별자리 코드를 추출하고 우리 zodiac ID로 매핑할 수 있다.
- 1위부터 12위까지 순위를 추출할 수 있다.
- 별자리별 일본어 운세 원문을 원문 그대로(문장부호 추가 없이) 추출할 수 있다.
- 행운의 장소 또는 아이템을 추출할 수 있다.
- 수집 결과를 Zod로 검증한다(12개 별자리, 순위 1~12 중복 없음, 날짜 유효성, 원문·행운아이템 비어있지 않음).

- 같은 날짜에 이미 12개 데이터가 있으면 수정하지 않고 건너뛴다(skip).
- 같은 날짜에 1~11개만 있는 애매한 상태는 수정·삭제하지 않고 오류로 처리한다.
- 실패해도 기존에 저장된 정상 데이터는 삭제하지 않는다(DELETE 미사용).

### M5. AI 학습 데이터 생성 및 배포 (PRD 개정: 한국어 세그먼트 생성 반영, 구현 완료)

- 출처가 홈 화면에 표시된다(PRD 개정: 상세 화면에서 홈 화면으로 이동). 원문 페이지 이동 링크는 현재 어느 화면에도 노출되지 않는다 — 상세 화면 개편으로 삭제된 뒤 대체 위치가 아직 결정되지 않은 상태이며, 별도 결정이 필요하다(PRD 개정: 확인된 격차).
- 데이터 수집 또는 AI 처리에 실패했을 때 잘못된 정보를 대신 표시하지 않는다.

- ~~초급 사용자에게 일본어 원문을 먼저 노출하면 진입 부담이 발생할 수 있다.~~ → 한국어 운세 본문을 상세 화면 기본 노출로 채택했다.
- ~~사용자 테스트와 Mixpanel 퍼널에서 이탈이 높을 경우 검토~~ → Mixpanel 데이터 없이 제품 판단으로 먼저 채택했으며, 아래 Mixpanel 계획으로 사후 검증한다.
- 온보딩/튜토리얼 도입은 여전히 검토 대상이며 아직 구현하지 않았다.

### Mixpanel 검증 계획 (PRD 개정: 여전히 보류 상태)
Mixpanel 연동은 **아직 구현하지 않았다.** 향후 연동 시 아래 이벤트 퍼널로 실제 사용성을 검증한다.

```
Fortune Opened
→ Vocabulary Checked
→ All Vocabulary Checked
→ Review Started
→ Vocabulary Saved
```

기존 퍼널의 `Translation Viewed` 이벤트는 삭제한다 — 별도의 "한국어 해석 보기" 액션이 더 이상 존재하지 않기 때문이다.

특정 기준값은 임의로 정하지 않는다. Mixpanel 연동 이후 실제 사용자 데이터를 확인한 뒤, 이 퍼널에서 관찰된 실제 이탈 지점을 근거로 판단한다.

### 확인된 격차 (별도 결정 필요)
- 원문 페이지 이동 링크가 상세 화면 개편으로 삭제된 뒤 대체 위치가 없다(24절 참고).
- 온보딩/튜토리얼 도입 여부는 아직 결정되지 않았다.
